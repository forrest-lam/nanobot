# 搜索聊天记录指南

本文档说明如何使用 QQChat MCP 工具搜索和分析聊天消息。

## 🎯 使用场景

**核心意图**：用户想要 **搜索消息内容** 或 **查找讨论过的话题**

**典型问题示例**：
- "谁提到过React" / "搜索包含'会议'的消息"
- "找'项目进度'相关的聊天"
- "技术群中李四发的关于React的消息"
- "最近3天有人找我吗"

## 优先级说明

- **高优先级方案（priority=3）**：最合适的方案，能直接查到数据，一步到位
- **中优先级方案（priority=2）**：可能需要中间步骤，查询结果需要进一步处理
- **低优先级方案（priority=1）**：兜底方案，需要多步处理，范围较大需要筛选

---

## 🟡 按关键词找聊天记录

**用户问**："谁提到过React" / "搜索包含'会议'的消息" / "找'项目进度'相关的聊天"

### 高优先级方案（priority=3）

**使用 search_messages 直接搜索**

**返回结果**：
- 按会话（uid）分组的消息片段（segments）
- 每个片段包含时间连续的消息
- 每条匹配消息自动带前后各3条上下文
- 包含 msgId、senderUid、senderName、content、timestamp

**优势**：
- 一步到位，直接获取包含关键词的消息
- 自动返回每条匹配消息的前后各3条上下文
- 时间连续的消息归为一个片段（segment）
- 支持多关键词OR搜索
- 可通过 senderUid 和 senderName 反推联系人身份

**适用场景**：
- 查找包含特定关键词的消息
- 回顾某个话题的讨论内容
- 通过聊天内容反推联系人或群组

### 中优先级方案（priority=2）

**通过 get_recent_messages 获取消息后客户端筛选**

**劣势**：
- 需要先知道会话uid（需要通过其他工具获取）
- 只能搜索最近的消息，较早的消息需要分页获取
- 需要客户端筛选，无法自动获取上下文
- 如果消息分散在多个会话，需要多次查询

**适用场景**：
- 已知会话uid，只需要搜索该会话的消息
- search_messages 不可用时的备选方案

### 低优先级方案（priority=1）

**通过 get_recent_chats 获取所有会话，逐个查询消息后筛选**

**劣势**：
- 需要获取大量消息数据，性能开销大
- 只能搜索最近会话，不活跃会话的消息找不到
- 客户端筛选工作量大，无上下文
- 可能需要多次分页查询

**适用场景**：
- search_messages 和 中优先级方案都不可用时
- 需要遍历所有会话进行全面搜索

## 高级技巧

### 利用消息片段理解上下文

**说明**：
- search_messages返回的每个segment都是时间连续的一组消息
- 自动包含匹配消息的前后各3条上下文
- 分析整个segment可以理解完整对话流程
- 例如：找到"会议"消息后，查看前后消息可了解会议时间、地点、参与者等

**示例**：
```
用户问："项目会议在哪开？"
→ search_messages({ keywords: ["会议"] })
→ 返回包含"会议"的消息片段（segment）
→ 从上下文消息中找到"3号会议室" 
```

### 参数组合实现精确筛选

**用户问**："上周张三在项目群发的所有消息"

**最优方案**：
- 步骤1：并行调用search_chats找"张三"和找"项目群"
- 步骤2：使用get_recent_messages，传入uids（项目群groupUid）、senderUids（张三uid）、date（上周时间）
- 优势：一次查询精确定位，无需客户端筛选

### 利用 senderName 反推联系人

**用户问**："小李最近说了什么？"（但不知道小李的真实昵称）

**步骤1**：使用search_messages全局搜索"小李"
**步骤2**：从返回消息中分析：
- 查看senderName包含"李"的发送者
- 查看@提及"小李"的消息
- 查看回复提到"小李"的消息的发送者

**步骤3**：使用get_profiles确认所有候选uid的详细资料
**步骤4**：根据昵称、备注匹配找到"小李"

### 善用 fromTime/toTime 参数查询时间范围

**用户问**："最近3天有人找我吗？"

**步骤1**：计算3天前的日期时间
**步骤2**：使用get_recent_messages，传入fromTime为3天前的时间（格式：yyyy-MM-dd HH:mm:ss），不传uids自动获取所有最近会话
**步骤3**：筛选最后一条消息不是自己发的会话

**fromTime/toTime参数格式**：`yyyy-MM-dd HH:mm:ss`，例如 `2024-01-01 00:00:00`，不传表示不限制
