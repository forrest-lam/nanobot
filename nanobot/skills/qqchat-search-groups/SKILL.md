# 搜索群组指南

本文档说明如何使用 QQChat MCP 工具搜索和定位群聊。

## 🎯 使用场景

**核心意图**：用户想要 **找某个群** 或 **查看群里的聊天内容**

**典型问题示例**：
- "找技术群" / "搜索项目组"
- "技术群最近在聊什么"
- "谁在群里最活跃"
- "总结一下项目群的讨论"

## 优先级说明

- **高优先级方案（priority=3）**：最合适的方案，能直接查到数据，一步到位
- **中优先级方案（priority=2）**：可能需要中间步骤，查询结果需要进一步处理
- **低优先级方案（priority=1）**：兜底方案，需要多步处理，范围较大需要筛选

---

## 🟢 按关键词找群

**用户问**："找技术群" / "搜索项目组" / "有个叫XX的群"

### 高优先级方案（priority=3）

**使用 search_chats 直接搜索**

调用方式：使用search_chats工具，传入keywords参数为群名数组

返回结果说明：
- groups数组包含匹配的群组
- 每个群有groupUid（群的唯一标识，纯数字字符串格式）、groupName（群名称）
- matchedMembers字段显示群内匹配关键词的成员列表

**优势**：
- 一步到位，直接获取群信息和groupUid
- 支持群名、群号搜索
- 返回matchedMembers显示群内匹配关键词的成员
- 群的uid是纯数字字符串，易于识别

**适用场景**：
- 用户明确提到某个群的名称
- 需要获取群的基本信息
- 需要查看群内成员

### 中优先级方案（priority=2）

**通过 get_recent_chats 从最近会话中筛选**

步骤1：调用get_recent_chats获取最近会话列表，设置limit为200
步骤2：从返回的会话列表中筛选chatType为2（群聊）的会话
步骤3：通过关键词或语义匹配，从群名称（peerName）中找到最有可能符合查询关键字的群聊

**劣势**：
- 只能找到最近有消息的群，不活跃的群找不到
- 需要进行语义匹配筛选，增加处理步骤
- 无法获取群成员信息

**适用场景**：
- search_chats不可用时
- 只需要在最近活跃群中查找
- 用户提到的群最近有聊天记录

### 低优先级方案（priority=1）

**通过 search_messages 搜索聊天记录中讨论过的群**

步骤1：使用search_messages搜索包含群名关键词的消息
步骤2：从消息上下文中查看是否有人讨论过这个群（例如："你加入XX群了吗"、"XX群里有人..."）
步骤3：如果消息来自群聊，该群的groupUid就可能是目标群；如果消息来自私聊，则从对话中推断可能的群名
步骤4：使用推断出的群名再次调用search_chats进行验证

**劣势**：
- 需要多步推断，过程复杂且可能不准确
- 依赖聊天记录中是否讨论过该群
- 如果没有相关讨论，无法找到

**适用场景**：
- 群名记不清，但知道某个群成员
- 群名记不清，但知道群里讨论的话题

## 高级技巧

### 利用 matchedMembers 字段

**用户问**："找一下有李四的技术群"

**最优方案**：
- 步骤1：使用search_chats，keywords传入"李四"
- 步骤2：查看返回的groups[].matchedMembers字段，说明"李四"在这些群里
- 优势：一次调用同时获取群列表和成员信息

### 通过联系人的 inGroups 字段找群

**用户问**："找一下和张三一起的那个项目群"

**最优方案**：
- 步骤1：使用search_chats找到"张三"
- 步骤2：从返回的contact.inGroups字段筛选包含"项目"关键词的群
- 优势：无需再次搜索群组，节省一次工具调用

### 参数组合实现精确筛选

**用户问**："上周张三在项目群发的所有消息"

**最优方案**：
- 步骤1：并行调用search_chats找"张三"和找"项目群"
- 步骤2：使用get_recent_messages，传入uids（项目群groupUid）、senderUids（张三uid）、date（上周时间）
- 优势：一次查询精确定位，无需客户端筛选

### 利用消息片段理解上下文

**说明**：
- search_messages返回的每个segment都是时间连续的一组消息
- 自动包含匹配消息的前后各3条上下文
- 分析整个segment可以理解完整对话流程
- 例如：找到"会议"消息后，查看前后消息可了解会议时间、地点、参与者等
