# 搜索联系人指南

本文档说明如何使用 QQChat MCP 工具搜索和定位联系人（好友）。

## 🎯 使用场景

**核心意图**：用户想要 **找某个人**、**查看与某人的聊天记录** 或 **查询联系人详细资料**

**典型问题示例**：
- "找一下张三" / "搜索老王"
- "和张三聊了什么" / "小李最近说了啥"
- "有哪些人给我发了消息"
- "最近我和谁聊得最多"
- "张三的QQ号是多少" / "小李在线吗" / "查一下老王的资料" / "李华的生日是什么时候"

## 优先级说明

- **高优先级方案（priority=3）**：最合适的方案，能直接查到数据，一步到位
- **中优先级方案（priority=2）**：可能需要中间步骤，查询结果需要进一步处理
- **低优先级方案（priority=1）**：兜底方案，需要多步处理，范围较大需要筛选

## 🔵 按关键词找人

**用户问**："找一下张三" / "搜索老王" / "有个叫小李的人"

### 高优先级方案（priority=3）

**使用 search_chats 直接搜索**

调用方式：使用search_chats工具，传入keywords参数为人名数组

返回结果说明：
- contacts数组包含匹配的联系人，每个联系人有uid（唯一标识）、nick（昵称）、remark（备注名）
- inGroups字段显示该联系人所在的所有群列表，每个群包含groupUid和groupName

**优势**：
- 一步到位，直接获取联系人信息和uid
- 支持昵称、备注、QQ号、拼音等多种搜索方式
- 返回结果包含inGroups字段，可直接看到该联系人在哪些群

**适用场景**：
- 用户明确提到某个人的名字、昵称或备注
- 需要获取联系人的基本信息
- 需要查看某人所在的群列表

### 中优先级方案（priority=2）

**通过 search_messages 搜索聊天内容反推联系人**

#### 执行步骤

**步骤 1：搜索消息**
- 调用 `search_messages` 搜索包含"张三"的消息

**步骤 2：快速检查 @ 提及**
- 检查返回的消息中是否有 @ 提及"张三"
- 如果有 `@张三` 格式，说明群里有人的群昵称就是"张三"
- ✅ 如果找到，直接使用这个名称搜索该群成员，流程结束

**步骤 3：深入分析聊天记录（仅在步骤 2 未找到时执行）**
- 3.1 统计提到"张三"最多的群
- 3.2 调用 `get_recent_messages` 获取该群的最近聊天记录

**步骤 4：多维度分析确认身份**

对获取到的聊天记录进行以下分析（可以组合使用）：

- **方式 A - 分析回复关系**
  - 查找回复了提到"张三"的消息的回复消息
  - 该回复消息的发送者很可能就是"张三"本人
  
- **方式 B - 语义匹配昵称**
  - 分析聊天成员的 `senderName`（发送者昵称）
  - 通过语义匹配（如包含"张"、"三"或其他相关特征）判断最可能的成员
  
- **方式 C - 批量拉取资料**
  - 调用 `get_profiles` 拉取所有 `senderUid` 的详细资料
  - 根据返回的昵称、备注、QQ号等信息确认最可能的"张三"

**劣势**：
- 需要多步推断，可能有误判
- 如果聊天记录中从未提到"张三"，则无法找到
- 推断过程复杂，准确性依赖上下文

**适用场景**：
- 口语化称呼（如"老王"）在search_chats中搜不到时
- 需要通过聊天内容反推联系人身份

### 低优先级方案（priority=1）

**通过 get_recent_chats 从最近会话中筛选**

步骤1：调用get_recent_chats获取最近会话列表，limit设为200
步骤2：从返回的会话列表中筛选包含"张三"昵称的联系人，需要遍历data数组，匹配peerName字段

**劣势**：
- 只能找到最近有聊天的联系人，不在最近会话中的联系人找不到
- 需要客户端遍历筛选，增加处理步骤
- 无法获取联系人所在的群列表信息

**适用场景**：
- search_chats和search_messages都不可用时
- 只需要在最近活跃会话中查找

## 高级技巧

### 利用 inGroups 字段快速定位

**用户问**："找一下和张三一起的那个项目群"

**最优方案**：
- 步骤1：使用search_chats找到"张三"
- 步骤2：从返回的contact.inGroups字段筛选包含"项目"关键词的群
- 优势：无需再次搜索群组，节省一次工具调用

### 使用 get_profiles 获取联系人详细资料

**用户问**："张三的QQ号是多少" / "小李在线吗" / "查一下老王的资料" / "这个人是谁" / "李华的生日是什么时候"

**步骤1**：通过 search_chats 或 search_messages 获取目标联系人的 uid
**步骤2**：调用 get_profiles，传入 uids 数组

**返回字段**：
- uid、uin（QQ号）、nick（昵称）、remark（备注名，仅好友有值）
- qid（自定义QQ ID）、longNick（个性签名）
- age（年龄）、sex（性别：0=未设置，1=男，2=女）、birthday（生日，YYYY-MM-DD）
- statusInfo.termDesc（终端描述，如"手机在线"）、statusInfo.customStatus（自定义状态：faceId + wording）

**注意**：uids 必须来自其他工具的返回结果，仅支持联系人资料，不支持群资料

### 参数组合实现精确筛选

**用户问**："上周张三在项目群发的所有消息"

**最优方案**：
- 步骤1：并行调用search_chats找"张三"和找"项目群"
- 步骤2：使用get_recent_messages，传入uids（项目群groupUid）、senderUids（张三uid）、date（上周时间）
- 优势：一次查询精确定位，无需客户端筛选
